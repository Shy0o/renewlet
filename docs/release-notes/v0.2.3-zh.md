# Renewlet v0.2.3

Renewlet v0.2.3 增加 Discord Webhook 和 PushPlus 通知渠道。手动管理周期订阅时，也可以只填写下一次续费日期，不必为了建档去补一个开始日期。

[English ->](./v0.2.3-en.md)

---

## 升级前请先读

Cloudflare 一键部署用户仍然从生成仓库运行 `Sync Renewlet Upstream` 升级，不要重新点一键部署按钮。旧生成仓库如果还没有这个入口，按 [Cloudflare 部署文档](https://github.com/zhiyingzzhou/renewlet/blob/main/docs/cloudflare-workers-deploy.zh-CN.md#旧一键部署用户) 一次性补上。

如果你的自动化、导出脚本或 Public API 调用直接读取 `startDate`，周期订阅现在可能没有开始日期；判断下一次扣费时间请使用下一次续费日期。

---

## 新功能

- 设置页新增 Discord Webhook 通知渠道，可配置 webhook、机器人名称和头像，并发送测试通知。
- 设置页新增 PushPlus 通知渠道，可配置 token、主题、模板和渠道，并发送测试通知。
- 手动周期订阅可以不填开始日期，只填写下一次续费日期。

---

## Bug 修复

- 取消浏览器通行密钥弹窗时，不再显示登录失败。
- 在验证码登录弹窗中取消通行密钥操作后，会保留当前验证码登录步骤。
