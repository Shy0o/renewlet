# Renewlet v0.2.3

Renewlet v0.2.3 adds Discord Webhook and PushPlus notification channels. Manual recurring subscriptions are also easier to add: if you only know the next renewal date, you no longer need to invent a start date.

[中文 ->](./v0.2.3-zh.md)

---

## Read Before Upgrading

Cloudflare one-click deploy users should still upgrade from the generated repository by running `Sync Renewlet Upstream`. Do not click the one-click deploy button again. If an older generated repository does not have this entry yet, add it once from the [Cloudflare deploy documentation](https://github.com/zhiyingzzhou/renewlet/blob/main/docs/cloudflare-workers-deploy.md#existing-one-click-deploy-users).

If your automation, export scripts, or Public API clients read `startDate` directly, recurring subscriptions may now have no start date. Use the next renewal date to decide when the next charge is due.

---

## New Features

- Add Discord Webhook notifications in Settings, with webhook URL, bot name, avatar, and test notification support.
- Add PushPlus notifications in Settings, with token, title, template, channel, and test notification support.
- Manual recurring subscriptions can be saved with a next renewal date and no start date.

---

## Bug Fixes

- Canceling the browser passkey prompt no longer shows a sign-in failure.
- Canceling passkey from an authenticator-code sign-in flow keeps the code step open instead of interrupting it.
